# NEXXOS MVP - Project Instructions

## Critical Architecture Decisions

### S3 Upload Strategy (IMPORTANT - June 2026)
- **Abacus hosted S3** uses temporary STS credentials (`ASIA*` prefix) that rotate frequently
- **Presigned URLs DO NOT WORK** from the mobile app because credentials expire between URL generation and mobile upload
- **Solution**: Use **proxy upload** (`POST /api/upload/direct`) — mobile sends file to backend via multipart/form-data, backend uploads to S3 with fresh credentials
- The presigned URL flow still exists for backwards compatibility but should NOT be used for mobile uploads on the Abacus dev backend
- **Direct S3 URLs** (e.g., `https://bucket.s3.region.amazonaws.com/key`) work fine for public files (reading/viewing)

### URL Resolution in Chat Messages
- Some old messages in DB have bare S3 keys (e.g., `37513/public/uploads/...`) instead of full URLs
- `resolveS3Url()` in `chat.service.ts` handles this by prepending the full S3 domain when URLs don't start with `http`

### Backend Environments
- **Dev (Abacus)**: `https://nexxos-api.abacusai.app` — uses `AWS_PROFILE=hosted_storage` (STS credentials)
- **QA (Render)**: `https://api.nexxos.app/` — uses long-lived IAM credentials from Supabase DB
- When switching between backends, the upload strategy matters: presigned URLs work on QA but NOT on Abacus dev
